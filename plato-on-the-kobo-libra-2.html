plato on the kobo libra 2

I caught the reading bug again

Published 2023-06-11, last edit 2023-06-12

For almost a year now I’ve been using plato on the kobo libra 2, an ereader that includes a backlight and a warmth scale (red light). The user experience has been good, here’s how setup works.

When you first get the device, you can plug it into your computer and mount it directly, everything is just owned by root: 

$ lsblk
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINTS
sda      8:48   0 232.9G  0 disk
├─sda1   8:49   0   550M  0 part /boot
├─sda2   8:50   0   225G  0 part /nix/store
│                                /
└─sdd3   8:51   0     4G  0 part [SWAP]
sdb      8:144  1  28.5G  0 disk

$ sudo mount /dev/sdj ~/kobo

To bypass the kobo account setup, you can exit the database on the device:

You might ask, why are we bypassing kobo account setup? because doing that means turning on the wifi. You do not want to turn on the wifi, because then automatic firmware updates will mess up your plato install and you’ll have to redo it (this may occur even if you have the automatic update setting turned off). 

$ sudo sqlite3 ~/kobo/.kobo/KoboReader.sqlite
sqlite> INSERT INTO user(UserID,UserKey) VALUES('1','');
sqlite> .quit

Now let’s install plato. Current releases can be found at the OP of this kobo forum post. The release is just a zip file that we will extract to ~/kobo.
